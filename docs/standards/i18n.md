# 国际化规范

本文档说明 IceZone Studio 的国际化（i18n）规范。

## 基础配置

### i18n 入口

`src/i18n/index.ts` - i18n 配置和初始化

### 语言文件

- `src/i18n/locales/zh.json` - 中文
- `src/i18n/locales/en.json` - 英文

### 使用方式

组件中统一使用 `useTranslation()` + `t('key.path')`：

```typescript
import { useTranslation } from '@/i18n'

function MyComponent() {
  const { t } = useTranslation()
  
  return (
    <div>
      <h1>{t('project.title')}</h1>
      <Button>{t('common.save')}</Button>
    </div>
  )
}
```

**禁止**：硬编码中英文文案

## Key 命名规范

### 模块化层级命名

使用点分隔的层级结构：

```json
{
  "project": {
    "title": "项目",
    "create": "创建项目",
    "delete": "删除项目"
  },
  "node": {
    "menu": {
      "uploadImage": "上传图片",
      "imageEdit": "图片编辑"
    }
  },
  "common": {
    "save": "保存",
    "cancel": "取消",
    "confirm": "确认"
  }
}
```

### Key 命名原则

1. **使用英文单词作为 key**，不使用中文句子
2. **key 必须稳定**，代码重构时不轻易改变
3. **key 必须可复用**，避免重复创建相似 key
4. **key 必须可检索**，使用有意义的命名

```json
✅ 正确示例：
{
  "node.create.success": "节点创建成功"
}

❌ 错误示例：
{
  "节点创建成功": "节点创建成功"
}
```

### 通用文案放置

通用文案优先放 `common.*`：

```json
{
  "common": {
    "save": "保存",
    "cancel": "取消",
    "delete": "删除",
    "edit": "编辑",
    "loading": "加载中...",
    "error": "错误",
    "success": "成功"
  }
}
```

页面专属文案放对应模块前缀：

```json
{
  "canvas": {
    "zoom": "缩放",
    "fit": "适应画布"
  }
}
```

## 新增文案流程

### 1. 先在 zh.json 增加新 key

```json
{
  "node": {
    "videoGen": {
      "title": "视频生成",
      "prompt": "描述",
      "duration": "时长"
    }
  }
}
```

### 2. 同步在 en.json 增加相同 key

```json
{
  "node": {
    "videoGen": {
      "title": "Video Generation",
      "prompt": "Prompt",
      "duration": "Duration"
    }
  }
}
```

**重要**：不要缺语言键，确保中英文 key 结构一致。

### 3. 代码里只引用 key

```typescript
<h3>{t('node.videoGen.title')}</h3>
<Input placeholder={t('node.videoGen.prompt')} />
```

**禁止**：写 fallback 字面量

```typescript
❌ 错误示例：
<h3>{t('node.videoGen.title') || '视频生成'}</h3>
```

## 节点默认标题 i18n

### 节点显示名定义

位置：`src/features/canvas/domain/nodeDisplay.ts`

### resolveNodeDisplayName 函数

```typescript
export function resolveNodeDisplayName(
  type: NodeType,
  data: any,
  t?: TFunction  // 可选，用于运行时语言切换
): string {
  if (t) {
    // 使用 i18n
    return t(`nodeDisplayName.${type}`)
  } else {
    // 降级到默认值
    return defaultNodeDisplayNames[type]
  }
}
```

### 节点组件中传入 t

```typescript
function ImageEditNode({ data }: NodeProps) {
  const { t } = useTranslation()
  
  const displayName = resolveNodeDisplayName('imageNode', data, t)
  
  return <div>{displayName}</div>
}
```

### i18n key 位置

统一放在 `nodeDisplayName.*`：

```json
{
  "nodeDisplayName": {
    "uploadNode": "上传图片",
    "imageNode": "图片编辑",
    "groupNode": "编组",
    "videoGenNode": "视频生成"
  }
}
```

## 动态值与复数

### 插值

使用插值语法传递动态值：

```typescript
// i18n
{
  "project.itemCount": "共 {{count}} 个项目",
  "user.welcome": "欢迎，{{name}}"
}

// 使用
t('project.itemCount', { count: 5 })  // "共 5 个项目"
t('user.welcome', { name: 'Alice' })  // "欢迎，Alice"
```

### 复数规则

使用 i18next 复数规则：

```json
{
  "file.count_one": "{{count}} 个文件",
  "file.count_other": "{{count}} 个文件"
}
```

```typescript
t('file.count', { count: 1 })  // "1 个文件"
t('file.count', { count: 5 })  // "5 个文件"
```

**禁止**：手写字符串拼接

```typescript
❌ 错误示例：
const text = count === 1 ? '1 个文件' : `${count} 个文件`

✅ 正确示例：
const text = t('file.count', { count })
```

### 格式化数字和时间

先格式化，再传给 `t`：

```typescript
const formattedDate = new Date(timestamp).toLocaleDateString(locale)
t('project.createdAt', { date: formattedDate })
```

## 语言切换

### 切换语言

```typescript
import { changeLanguage } from '@/i18n'

function LanguageSwitcher() {
  const handleChange = (lang: 'zh' | 'en') => {
    changeLanguage(lang)
  }
  
  return (
    <Select value={currentLang} onChange={handleChange}>
      <option value="zh">中文</option>
      <option value="en">English</option>
    </Select>
  )
}
```

### 持久化

语言选择存储到 `localStorage`：

```typescript
localStorage.setItem('i18n-lang', lang)
```

应用启动时自动读取：

```typescript
const savedLang = localStorage.getItem('i18n-lang')
if (savedLang) {
  changeLanguage(savedLang)
}
```

## 最低验证

### 检查 key 是否泄露

切换中英文后，不应出现未翻译 key 直接显示（例如显示 `project.title`）。

如果出现，说明：
1. 语言文件缺少该 key
2. 代码中未使用 `t()` 函数

### 检查 key 是否齐全

新增 key 时，确保中英语言包均存在：

```bash
# 检查 key 数量是否一致
jq 'keys | length' src/i18n/locales/zh.json
jq 'keys | length' src/i18n/locales/en.json
```

### 检查关键文案

关键按钮、提示、错误文案在两种语言下都：
- 可读
- 不截断
- 不重叠

## 特殊场景

### 富文本

如果需要在翻译中插入 HTML 或组件：

```typescript
// i18n
{
  "terms.accept": "我同意<link>服务条款</link>"
}

// 使用 Trans 组件
import { Trans } from 'react-i18next'

<Trans
  i18nKey="terms.accept"
  components={{
    link: <a href="/terms" />
  }}
/>
```

### 长文本

对于长段落，可以使用数组：

```json
{
  "welcome": {
    "paragraph1": "欢迎使用 IceZone Studio。",
    "paragraph2": "这是一个基于节点画布的 AI 创作工具。"
  }
}
```

```typescript
<div>
  <p>{t('welcome.paragraph1')}</p>
  <p>{t('welcome.paragraph2')}</p>
</div>
```

## 命名空间

对于大型项目，可以使用命名空间隔离：

```typescript
// src/i18n/locales/zh/common.json
{
  "save": "保存",
  "cancel": "取消"
}

// src/i18n/locales/zh/canvas.json
{
  "zoom": "缩放",
  "fit": "适应画布"
}

// 使用
t('common:save')
t('canvas:zoom')
```

## 测试

### 测试翻译是否存在

```typescript
describe('i18n keys', () => {
  it('should have all required keys in both languages', () => {
    const zhKeys = Object.keys(zhTranslations)
    const enKeys = Object.keys(enTranslations)
    
    expect(zhKeys.sort()).toEqual(enKeys.sort())
  })
})
```

### 测试组件多语言

```typescript
describe('LanguageSwitcher', () => {
  it('should render text in Chinese', () => {
    changeLanguage('zh')
    render(<MyComponent />)
    
    expect(screen.getByText('保存')).toBeInTheDocument()
  })
  
  it('should render text in English', () => {
    changeLanguage('en')
    render(<MyComponent />)
    
    expect(screen.getByText('Save')).toBeInTheDocument()
  })
})
```
